## UI Implementation Notes

### Collapsible Sections
- Use accordion/collapse components
- Save collapse state in local storage
- Show badge with count when collapsed (e.g., "âš¡ 2 assets, 1 dependency")

### Validation
- Name is required
- Task Category (L1) is required
- Service Type (L2/L3) is required
- State is required
- All other fields optional
- Validate JSON in Custom Data field
- Validate date ranges (end >= start)

### Smart Defaults
- State defaults to "assigned"
- Generate UUIDs automatically
- Context array only created if scheduling or relationships added
- workUnit only included if participant selected
- taskGroup only included if group selected

### Search/Autocomplete
For participant/asset/task selectors:
- Fuzzy search by name
- Show type/role in dropdown
- Display selected items as chips/badges
- Allow removal of selected items

### Date/Time Handling
- Use date pickers for dates
- Use time pickers for times
- Combine into ISO 8601 format: "YYYY-MM-DDTHH:mm:ssZ"
- Show in user's timezone but store as UTC
