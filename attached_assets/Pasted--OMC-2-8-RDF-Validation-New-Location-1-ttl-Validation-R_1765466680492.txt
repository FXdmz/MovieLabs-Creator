Note how the **same location data** uses different property names in each format:

```json
{
  "address": {
    "street": "2231 Rue Quesnel",
    "locality": "Montreal",
    "region": "Quebec",
    "postalCode": "H3J 2A9",
    "country": "CA"
  },
  "coordinates": {
    "latitude": 45.487343,
    "longitude": -73.57547
  }
}
```

### RDF Format (from New_Location__1_.ttl)
```turtle
_:address_1
    rdf:type omc:Address ;
    omc:hasStreetNumberAndName "2231 Rue Quesnel" ;
    omc:hasCity "Montreal" ;
    omc:hasPostalCode "H3J 2A9" ;
    omc:hasCountry "CA" .

_:coordinates_2
    rdf:type omc:LatLon ;
    omc:hasLatitude "45.487343"^^xsd:decimal ;
    omc:hasLongitude "-73.57547"^^xsd:decimal .
```

**Key Differences:**
1. JSON has `region` (Quebec) - RDF doesn't support it
2. JSON uses simple names - RDF uses explicit predicates
3. Both are valid in their respective formats!
